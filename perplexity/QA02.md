## ② 想讓結果更精準：拆成 5 個「子 Prompt」

### A. 只做「104 vs 1111 欄位對照表」
請查找並整理 104 與 1111 線上履歷的欄位與功能差異，輸出：

1. 共同欄位清單
2. 104 獨有欄位/功能
3. 1111 獨有欄位/功能（含附件作品/上傳連結等）
4. 我該預先準備的素材清單（照片、聯絡方式、作品集連結、檔案格式等）
    請用表格呈現並附來源（優先官方/官方說明頁）。

```

```txt
### B. 只做「三科系履歷寫法策略」

我是【資訊/管理/文創設計】科系學生，目標職位【】。請提供：

* 這個科系最有效的履歷排序（哪些段落要放前面）
* 10 個可量化成果句模板（含動詞、量化方式、例句）
* 3 段「同一個經歷」如何改寫成不同職位/不同科系角度的示範
* 常見地雷與修正方式

```
### C. 只做「證書（Coursera/eWant）怎麼放」

我有【Coursera/eWant/其他】課程證書【列 3~10 個】。請告訴我：

* 哪些適合放在履歷的「證照/課程」區、哪些應該放到作品集或 LinkedIn
* 每張證書要怎麼寫才不會像「湊字數」：請給條目模板 + 完整示範 3 則
* LinkedIn 上應該放在 Licenses & Certifications 還是 Courses？何時放 Featured？請給決策規則
* 如果證書沒有 credential ID/URL，要怎麼處理敘述
```

```txt
### D. 只做「作品集架構」

請針對【資訊/管理/文創設計】提供作品集呈現方案：

* 作品挑選規則（用 80/20）

* 一頁 Case Study 模板（Problem / Process / Solution / Result / Reflection）
* 如果我要投【某職缺】，哪些作品要放最前面？請給排序策略
```

   的寫作框架、可套用模板、優秀公開範例來源，並列出新鮮人最常犯的 10 個錯誤與修正。

```txt
## ③ 小技巧：讓 Perplexity 產出更像「作業/報告」而不是碎片回答

你可以在任何 Prompt 最後加這段「輸出規格」：

> 輸出規格：
>
> * 先給「一頁總覽」（清單 + 表格）再給詳細說明
> * 每個段落最後加「我下一步要做什麼」
> * 所有模板都要用可複製的 markdown 區塊
> * 引用要標出來源與更新日期（若可取得）
```
